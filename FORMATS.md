# Форматы сохранения данных

## Зачем нужны оба формата?

### JSON (`rb_articles.json`)
**Преимущества:**
- ✅ Сохраняет структуру данных (списки остаются списками)
- ✅ Удобен для программной обработки (Python, JavaScript)
- ✅ Легко парсить и обрабатывать
- ✅ Подходит для загрузки в NoSQL БД (MongoDB, Elasticsearch)

**Недостатки:**
- ❌ Неудобен для Excel
- ❌ Больше размер файла (из-за форматирования)

**Пример использования:**
```python
import json
with open('rb_articles.json', 'r', encoding='utf-8') as f:
    articles = json.load(f)
    # Списки остаются списками
    companies = articles[0]['companies']  # ['Яндекс', 'Тинькофф']
```

### CSV (`rb_articles.csv`)
**Преимущества:**
- ✅ Удобен для Excel и Google Sheets
- ✅ Легко анализировать в pandas
- ✅ Меньше размер файла
- ✅ Удобен для импорта в SQL БД

**Недостатки:**
- ❌ Теряет структуру (списки превращаются в строки)
- ❌ Сложнее обрабатывать вложенные данные

**Пример использования:**
```python
import pandas as pd
df = pd.read_csv('rb_articles.csv')
# Списки теперь строки: "Яндекс; Тинькофф"
companies = df['companies'].iloc[0].split('; ')  # Нужно парсить
```

## Время сохранения

**Важно:** Сохранение занимает очень мало времени по сравнению со скрапингом!

| Количество статей | JSON | CSV | Итого |
|-------------------|------|-----|-------|
| 1,000 | ~0.2 сек | ~0.3 сек | ~0.5 сек |
| 10,000 | ~1-2 сек | ~2-3 сек | ~3-5 сек |
| 50,000 | ~5-10 сек | ~10-15 сек | ~15-25 сек |

**Сравнение:**
- Скрапинг 10,000 статей: ~1-3 часа
- Сохранение в оба формата: ~3-5 секунд
- **Экономия времени: 0.03%** - практически незаметно!

## Рекомендации

### Если нужен только один формат:

**Для программной обработки:**
```python
scraper.save_to_json('rb_articles.json')  # Только JSON
```

**Для анализа в Excel/pandas:**
```python
scraper.save_to_csv('rb_articles.csv')  # Только CSV
```

### Если нужны оба формата:

Оставляйте как есть - время сохранения пренебрежимо мало, а вы получаете:
- JSON для программной обработки
- CSV для анализа в таблицах

## Размер файлов

Примерные размеры для 10,000 статей:
- JSON: ~50-100 MB (с форматированием)
- CSV: ~30-50 MB (более компактный)

Оба формата занимают разумное место на диске.
